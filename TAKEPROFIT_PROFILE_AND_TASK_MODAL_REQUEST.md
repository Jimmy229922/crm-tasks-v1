# طلب تنفيذ: Profile Navigation + Task Details Coming Soon Modal

## الهدف
تنفيذ سلوكين واضحين داخل صفحة CRM Task Management في مشروع Take Profit:
1. عند الضغط على أيقونة Profile داخل صف المهمة -> يتم فتح صفحة البروفايل.
2. عند الضغط على زر/أيقونة Task Details -> يظهر مودال في نفس الشاشة مكتوب فيه: Coming Soon.

## النطاق المطلوب
- التعديل داخل موديول CRM فقط.
- لا حاجة لربط Backend حقيقي حاليا.
- الاعتماد على Mock Data أو البيانات المتاحة حاليا.

## السلوك المطلوب بالتفصيل

### 1) Profile Icon Click Behavior
- كل صف في الجدول يحتوي أيقونة Profile.
- عند الضغط عليها:
  - يتم الانتقال إلى Route صفحة البروفايل الخاصة بالعميل.
  - يتم تمرير `clientId` الصحيح من بيانات الصف.
- مثال متوقع لمسار البروفايل:
  - `/crm/tasks-v1/profile/:clientId`

### 2) Task Details Button/Icon Click Behavior
- كل صف في الجدول يحتوي زر أو أيقونة Task Details.
- عند الضغط عليها:
  - لا يتم فتح صفحة جديدة.
  - يظهر Modal Overlay فوق نفس الصفحة.
  - محتوى المودال يكون واضح ومباشر:
    - العنوان: Coming Soon
    - وصف بسيط: Task details will be available soon.
- المودال لازم يدعم الإغلاق عبر:
  - زر Close داخل المودال.
  - الضغط على الخلفية (Backdrop).
  - زر Escape من الكيبورد.

## متطلبات واجهة وتجربة استخدام
1. المودال يلتزم بنفس الـ Dark Theme.
2. يكون المودال في المنتصف، وخلفية معتمة خفيفة.
3. لا يكسر التخطيط الأساسي للصفحة.
4. لا يؤثر على وظيفة البحث/الفلتر/الـ pagination.

## متطلبات تقنية
1. الالتزام بنفس معايير المشروع الحالية داخل Take Profit (Routing, State, UI conventions).
2. عدم إضافة مكتبات جديدة إذا يمكن تنفيذ المطلوب بالأدوات الموجودة.
3. إبقاء الكود منظم وقابل للتوسعة لاحقا عندما يبدأ ربط Backend.

## معايير القبول (Acceptance Criteria)
- الضغط على أيقونة Profile ينقلني للبروفايل الصحيح لكل صف.
- الضغط على أيقونة/زر Task يفتح مودال Coming Soon داخل نفس الصفحة.
- يمكن إغلاق المودال بالطرق الثلاث (Close button, Backdrop, Escape).
- لا توجد أخطاء Console أثناء التفاعل.
- لا يوجد كسر بصري في الثيم الداكن.

## المطلوب تسليمه بعد التنفيذ
ارجع بملف Markdown باسم:

TAKEPROFIT_PROFILE_AND_TASK_MODAL_DELIVERY_REPORT.md

ويحتوي على:
1. ما تم تنفيذه بالضبط.
2. الملفات التي تم تعديلها.
3. كيف تم تمرير `clientId` للبروفايل.
4. كيف تم تنفيذ Coming Soon Modal.
5. أي ملاحظات مهمة للمرحلة القادمة (عند ربط Backend).
