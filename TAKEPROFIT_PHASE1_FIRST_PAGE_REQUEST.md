# طلب تنفيذ المرحلة الأولى داخل مشروع Take Profit

## الهدف
تنفيذ صفحة واحدة فقط من موديول CRM في هذه المرحلة:
- صفحة Task Management كما هي في التصميم المرفق.
- وعدم تنفيذ باقي الصفحات الآن.

## تنبيه مهم جدا
التنفيذ المطلوب هنا داخل مشروع Take Profit فقط.
لا يتم تعديل مشروع CRM الحالي عندي.

## النطاق المطلوب حاليا
1. تنفيذ صفحة Task Management فقط (الصفحة الأولى).
2. أي دخول على صفحة البروفايل يظهر شاشة Coming Soon.
3. أي دخول على تفاصيل المهمة أو فتح مودال التفاصيل يظهر Coming Soon.
4. لا ربط Backend فعلي حاليا. استخدم بيانات Mock فقط.

## شكل الصفحة المطلوب
نفذ نفس تصميم الصورة المرفقة بدقة عالية:
- ثيم داكن بالكامل Dark Theme.
- شريط علوي فيه:
  - Badge بعنوان CRM Suite.
  - عنوان رئيسي Task Management.
  - زر Create New Task في الجهة اليمنى.
- كارت رئيسي بعنوان All Tasks مع عدد المهام.
- Search bar كبيرة مع زر Search.
- فلتر Task Type بجانب البحث.
- جدول المهام بنفس الروح البصرية:
  - CLIENT INFO
  - TASK TYPE
  - STATUS
  - ASSIGNEE
  - CREATED
  - DUE DATE
  - ACTIONS

## قواعد واجهة مهمة
1. عمود CLIENT INFO يكون خلية واحدة تجمع:
- اسم العميل
- البريد الإلكتروني
- أفاتار بحروف الاسم

2. عمود ASSIGNEE يحتوي:
- أفاتار
- اسم الموظف
- المسمى الوظيفي

3. عمود ACTIONS يحتوي 3 أيقونات مثل التصميم.

4. حالات STATUS تظهر كـ badges واضحة (مثل Pending و In Progress) بألوان متسقة مع الثيم الداكن.

5. Task Types تظهر كـ badges ملونة ومقروءة.

## السلوك المطلوب الآن
1. البحث يعمل Frontend على بيانات Mock.
2. فلتر Task Type يعمل Frontend على بيانات Mock.
3. Pagination تعمل Frontend.
4. الضغط على أي زر يخص Profile أو Task Details يعرض Coming Soon (صفحة أو مودال حسب بنية المشروع عندكم).
5. زر Create New Task حاليا يمكن يكون UI فقط بدون ربط Backend.

## متطلبات التوافق مع معايير Take Profit
1. الالتزام بنفس Stack وأنماط المشروع لديكم (Architecture + Styling + State/Data conventions).
2. الالتزام بنظام i18n الموجود لديكم.
3. الحفاظ على الثيم الداكن الحالي وعدم إدخال أي Light Theme.
4. عدم إدخال مكتبات جديدة إلا لو ضرورة واضحة ومبررة.

## المطلوب تسليمه بعد التنفيذ
ارجع لي بملف Markdown باسم:

TAKEPROFIT_CRM_PHASE1_DELIVERY_REPORT.md

ويحتوي على:
1. ملخص ما تم تنفيذه.
2. قائمة الملفات التي تم تعديلها.
3. ما تم تنفيذه في صفحة Task Management بالتفصيل.
4. كيف تم تطبيق Coming Soon للبروفايل والتفاصيل.
5. أي فروق عن التصميم المرفق (إن وجدت) مع السبب.
6. نقاط جاهزية المرحلة التالية لربط Backend.

## ملاحظات إضافية
- المطلوب في هذه المرحلة هو واجهة الصفحة الأولى فقط بشكل متقن.
- الباك اند الحقيقي سيتم لاحقا من طرفنا.
- أي قرار تنفيذي يؤثر على قابلية الربط لاحقا يجب توضيحه في التقرير.
